Key Terms

FBA: Flux Balance Analysis (FBA) of metabolic models determines the flow of metabolites through the model in order to optimize an objective function (e.g. biomass production)

FVA: Flux variability analysis (FVA) (Mahadevan and Schilling, 2003) shows the range of fluxes that each reaction can achieve. These ranges are found by maximizing and minimizing of each flux individually.

Fluxes/flux ranges: “Fluxes” are a way of describing the flow of metabolites through a model. A reaction’s ‘flux range’ has an upper and lower bound which characterizes its production level.

Directionality of fluxes: Positive flux range means that the reaction is running in the forward direction, negative flux range means the reaction is running in the reverse direction, a range containing both positive and negative values means it is reversible. If the flux range contains 0, the reaction is nonessential.

Overlapping/nonoverlapping ranges: We discuss flux range overlap when we are comparing two models- for example, wild type vs. N2 fixing WM1788. Flux ranges that are nonoverlapping for the same reaction indicate that the organism is using a different metabolic pathway.

“Cycling”: This means that the flux range is hitting the maximum bound prescribed by our objective function. Generally when reactions are cycling, they don’t hold as much useful information for our analysis (these reactions shouldn’t behave this way in vivo).

Biomass: The biomass reaction is an artificial reaction created in our model to measure the organism growth rate. It is made by using experimental results about what the components of biomass are, and then scaled so it matches the exponential growth rate of the organism. So basically biomass = cell growth.

‘Fixed’ Biomass: In order to normalize our results for ATP production/other metabolite production, we need to set biomass of our strain variations to the same value. For example, N2 fixing WM1788 obviously has a much lower biomass than the wild type. But, in order to make comparisons between the two models, we need to set biomass to the same value.

‘Max’ Biomass: The maximum growth for a strain.

ATP Maintenance Reaction: ATP -> ADP + P. We maximize this reaction to determine the ATP production level for a strain. Similarly, when we talk about “maximizing flavodoxin”, we created a flavodoxin exporter which we are maximizing in our programs to measure how much flavodoxin the strain can produce.

GAMS: A programming language which we use to run FBA and FVA.

Loop statement: A statement that iteratively runs through a list of variables (i.e. a list of reactions).

Python: We use Python to generate input files for GAMS, and sometimes to iteratively run GAMS.

BioCyc, EcoCyc, Model SEED: These are online databases that have information about all the reactions in our model. They are helpful for learning about ∆G of a reaction (if we think there is a discrepancy in our model) or generally about what a reaction is/does/what pathway it is a part of.
